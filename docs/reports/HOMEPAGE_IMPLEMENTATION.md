# Homepage v2 Implementation Report

**Date:** 2026-06-27  
**Branch:** main  
**Build:** Static-first Phase 1

---

## Summary

Homepage v2 replaces the previous multi-section tutorial-style homepage with a four-section premium product experience following the approved blueprint. The design principle: tell one story — *"I can build amazing things with ESP32."*

---

## Sections Implemented

### Section 1 — The Declaration (Hero)
**Blueprint headline:** "Build things that exist in the real world."

- Full-viewport dark hero (`#0A0E1A`) with radial gradient atmosphere
- Poppins 800, `clamp(2.5rem, 5.5vw, 5rem)`, letter-spacing `-0.035em`
- Single CTA: "Start Building →" — white button on dark, inverted hierarchy
- Right visual: custom ESP32 board SVG with `feGaussianBlur` glow filter
- Three floating data cards (Temperature / Wi-Fi / LED) with dark-glass style, animated with `v2float` keyframes
- `-webkit-backdrop-filter` + `backdrop-filter` for Safari compatibility

### Section 2 — Proof of Possibility
**Blueprint headline:** "What engineers build here."

- Full-viewport-width horizontal scroll panels (`scroll-snap-type: x mandatory`)
- Four showcase panels, each `min-height: 72vh`:
  - **Weather Station** — blues gradient, DHT22 + OLED SVG illustration
  - **Smart Irrigation** — greens gradient, plant + soil sensor SVG
  - **Motion Security Alert** — reds gradient, shield + PIR radar SVG
  - **Home Climate Automation** — purples gradient, house + thermostat SVG
- Each panel has a radial glow (`filter: blur(90px)`) behind illustration
- Dot indicator updates on scroll (JS: `scroll` event → `scrollLeft` ratio)
- Drag-to-scroll on desktop (mousedown/mousemove/mouseup)
- Mobile: vertical stack, scroll-snap disabled, dots hidden

### Section 3 — The Engine
**Blueprint headline:** "Missions, not modules."

- Pure white background — stark contrast bookend from dark sections
- Three-pillar grid (`LEARN / BUILD / SHIP`) separated by subtle CSS `::before`/`::after` dividers
- Mission arc: 12-node SVG-free CSS track showing Mission 01–12
  - Milestone nodes at positions 1, 4, 8, 12
  - Live missions (01, 02, 03) are active links to actual guide pages
  - Hover: dot scales up, label activates in primary color
- No CTA in this section — it is informational by design

### Section 4 — The Invitation
**Blueprint headline:** "Your first project ships in two hours."

- Dark background returns (`#0A0E1A`) — visual bookend with Section 1
- Centered layout, max-width 600px
- Single CTA: "Start Mission 01 — Free →" — primary blue button
- Ghost text: "No account required to begin." (`rgba` 25% opacity)
- CTA links directly to `/guides/blink-led-esp32.html`

## CSS Architecture

All v2 styles use the `.v2-` prefix to avoid collisions with existing page styles. No existing CSS rules were modified.

New CSS blocks added to `style.css`:
- `.v2-declaration` + children — hero section
- `.v2-proof` + `.v2-showcase-*` — showcase track & panels
- `.v2-engine` + `.v2-pillar-*` + `.v2-arc-*` — engine section
- `.v2-invitation` + `.v2-btn-invitation` + `.v2-no-account` — CTA section
- `@media (max-width: 960px)` — tablet responsive
- `@media (max-width: 768px)` — mobile responsive
- `@media (prefers-color-scheme: dark)` — full dark mode via CSS variable overrides

### Dark Mode Strategy
Dark mode overrides `:root` CSS variables (`--bg`, `--surface`, `--text`, `--border`, etc.) so all existing components that use variables get dark mode automatically. Specific component overrides added for: nav, buttons, path cards, component cards, mission cards, search overlay, footer.
