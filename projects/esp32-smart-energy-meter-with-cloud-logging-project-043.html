ESP32 Energy Meter with Live Telegram Bot Updates with Telegram Bot

Energy Monitoring · Intermediate · 8 min read · Project

Learn how Voltage Sensor signal data on GPIO36 triggers Current Sensor control for reliable Voltage Node operation.

This tutorial walks through energy meter with live telegram bot updates with telegram bot — wiring, firmware, and a quick bench test.

Expect about 10 minutes to wire the board, flash firmware, and verify Serial output.

Built for voltage node: extend later with SPI Display if you need remote visibility.

Project Overview

A beginner-friendly Energy Monitoring circuit where Voltage Sensor signal feeds the ESP32 and Current Sensor control handles the physical action.

  • Voltage Sensor signal connects to GPIO36 (ADC input)
  • Threshold is set to 2822 in firmware (adjust after calibration)
  • Current Sensor control is driven on GPIO39 when the condition is met
  • The main loop samples every 637 ms while powered

How It Works

01

Read

The ESP32 samples the input sensor on GPIO36, producing a raw reading.

02

Compare

That reading is checked against a threshold configured in the sketch.

03

Decide

When the condition is met, the ESP32 sets GPIO39 high; otherwise it stays low.

04

Act

The ESP32 drives Current Sensor control on GPIO39 high while the condition stays true, then releases it.

Code

Upload the following sketch to your ESP32 board using the Arduino IDE. Adjust pins and threshold for your hardware.

esp32_smart_energy_meter_with_cloud_logg.ino
// ESP32 Energy Meter with Live Telegram Bot Updates with Telegram Bot
const int SENSOR_PIN = 36;
const int OUTPUT_PIN = 39;
const int THRESHOLD = 3029;
const int SAMPLE_DELAY_MS = 637;

void setup() {
  Serial.begin(115200);
  pinMode(OUTPUT_PIN, OUTPUT);
  pinMode(SENSOR_PIN, INPUT);
  Serial.println("Starting: energy meter with live telegram bot updates with telegram bot");
  Serial.print("Threshold: ");
  Serial.println(THRESHOLD);
}

void loop() {
  int reading = analogRead(SENSOR_PIN);
  Serial.print("Voltage Sensor signal: ");
  Serial.println(reading);
  bool active = reading > THRESHOLD;
  digitalWrite(OUTPUT_PIN, active ? HIGH : LOW);
  delay(SAMPLE_DELAY_MS);
}
