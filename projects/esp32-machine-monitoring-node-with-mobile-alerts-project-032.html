ESP32 Machine Monitoring Node - GPIO Interrupt Build v2

Industrial Automation · Beginner · Project

Build an ESP32 based machine monitoring node that monitors data using a current sensor and provides reliable automation for real-world use.

This tutorial explains how to build ESP32 Machine Monitoring Node - GPIO Interrupt Build v2 from wiring to working firmware.

In this project, the ESP32 reads Current Sensor signal on GPIO35 and drives Relay Output control on GPIO19 when the threshold is crossed.

ESP32 Machine Monitoring Node - GPIO Interrupt Build v2 is designed as a practical industrial automation build you can test on a breadboard and later expand with Wi-Fi or cloud features.

Project Overview

Build an ESP32 based machine monitoring node that monitors data using a current sensor and provides reliable automation for real-world use.

  • The ESP32 connects to your circuit and reads the input sensor;
  • Sensor values are compared against a threshold in the sketch;
  • When the condition is met, the output device is activated on GPIO19;
  • The loop repeats while the board is powered.

How It Works

01

Read

The ESP32 samples the input sensor on GPIO35, producing a raw reading.

02

Compare

That reading is checked against a threshold configured in the sketch.

03

Decide

When the condition is met, the ESP32 sets GPIO19 high; otherwise it stays low.

04

Act

ESP32 continuously reads the current sensor. When the reading crosses the configured threshold, it activates the relay output and updates the local output for the user.

Code

Upload the following sketch to your ESP32 board using the Arduino IDE. Adjust pins and threshold for your hardware.

esp32_machine_monitoring_node_with_mobil.ino
// ESP32 Machine Monitoring Node - GPIO Interrupt Build v2
const int sensorPin = 35;
const int outputPin = 19;
void setup() { Serial.begin(115200); pinMode(outputPin, OUTPUT); }
void loop() { int value = analogRead(sensorPin); Serial.println(value); digitalWrite(outputPin, value > 2000 ? HIGH : LOW); delay(1000); }
