ESP32 Distance Monitoring System

Project tutorial sections

SensorBeginnerIntermediateAdvanced

Measure distance with ultrasonic or ToF sensors on ESP32 — beginner breadboard builds through OLED feedback and tuned alerts.

Overview

This beginner stage of ESP32 Distance Monitoring System uses an HC-SR04 ultrasonic sensor. The ESP32 sends a trigger pulse, measures the echo pulse width, converts it to centimeters, and sounds a buzzer when an object is closer than the configured limit.

Components
  • ESP32 DevKit (30-pin)
  • Breadboard
  • Jumper wires
  • USB cable (5 V power)
  • HC-SR04 ultrasonic distance sensor
  • 10 kOhm resistor
  • 20 kOhm resistor
  • Active buzzer

Arduino Code
esp32-distance-monitoring-system_beginner.ino
// ESP32 Distance Monitoring System - HC-SR04 distance monitor
const int TRIG_PIN = 5;
const int ECHO_PIN = 18;
const int BUZZER_PIN = 25;
const int DISTANCE_LIMIT_CM = 20;
const int SAMPLE_DELAY_MS = 250;
long readDistanceCm() {
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);
  long duration = pulseIn(ECHO_PIN, HIGH, 30000);
  if (duration == 0) return -1;
  return duration / 58;
}

void setup() {
  Serial.begin(115200);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  digitalWrite(BUZZER_PIN, LOW);
}

void loop() {
  long distanceCm = readDistanceCm();
  Serial.print("Distance cm: ");
  Serial.println(distanceCm);
  bool tooClose = distanceCm > 0 && distanceCm < DISTANCE_LIMIT_CM;
  digitalWrite(BUZZER_PIN, tooClose ? HIGH : LOW);
  delay(SAMPLE_DELAY_MS);
}
How It Works
01

GPIO5 sends a 10 microsecond pulse to the HC-SR04 TRIG pin.

02

The HC-SR04 drives ECHO high for the sound travel time; GPIO18 reads that pulse through a 10 kOhm / 20 kOhm divider.

03

The sketch converts pulse width to distance in centimeters.

04

GPIO25 drives the buzzer when distance is below DISTANCE_LIMIT_CM.

Applications
  • Distance monitoring with an HC-SR04 ultrasonic sensor
Troubleshooting

Serial Monitor shows no distance

Check TRIG on GPIO5, ECHO on GPIO18, common ground, and 115200 baud.

Readings jump or show zero

Keep the target flat, stay within the sensor range, and verify the ECHO divider uses 10 kOhm and 20 kOhm resistors.

Buzzer never sounds

Move an object closer than DISTANCE_LIMIT_CM and verify buzzer + is on GPIO25.

Upgrades
  • Publish readings to an MQTT broker or Home Assistant

Arduino Code
esp32-distance-monitoring-system_advanced.ino
// ESP32 Distance Monitoring System - HC-SR04 distance monitor
#include <WiFi.h>
#include <WebServer.h>

const int TRIG_PIN = 5;
const int ECHO_PIN = 18;
const int BUZZER_PIN = 25;
const int DISTANCE_LIMIT_CM = 20;
const int SAMPLE_DELAY_MS = 250;
const char* WIFI_SSID = "YOUR_SSID";
const char* WIFI_PASS = "YOUR_PASSWORD";
WebServer server(80);

long readDistanceCm() {
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);
  long duration = pulseIn(ECHO_PIN, HIGH, 30000);
  if (duration == 0) return -1;
  return duration / 58;
}

void setup() {
  Serial.begin(115200);
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  digitalWrite(BUZZER_PIN, LOW);
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  while (WiFi.status() != WL_CONNECTED) delay(300);
  server.on("/", []() {
    server.send(200, "text/plain", "Distance: " + String(readDistanceCm()) + " cm");
  });
  server.begin();
  Serial.println(WiFi.localIP());
}

void loop() {
  server.handleClient();
  long distanceCm = readDistanceCm();
  Serial.print("Distance cm: ");
  Serial.println(distanceCm);
  bool tooClose = distanceCm > 0 && distanceCm < DISTANCE_LIMIT_CM;
  digitalWrite(BUZZER_PIN, tooClose ? HIGH : LOW);
  delay(SAMPLE_DELAY_MS);
}

What You Learn
01

ESP32 Distance Monitoring System starts with a real input: HC-SR04 ultrasonic sensor. The first learning goal is to prove the ESP32 can read that signal reliably before anything is automated.

02

The second goal is decision logic. The sketch compares the live reading with a threshold, then explains why thresholds need testing instead of guessing.

03

The final goal is a controlled output: buzzer alert. You learn how a sensor projects project turns sensor data into a visible or useful action.

Safety and Accuracy Notes

Editorial accuracy

ESP32 Distance Monitoring System is written as an educational build. Last updated: 2026-06-18. Pin choices, code structure, and troubleshooting steps are selected so beginners can test the circuit on a bench before moving to a permanent enclosure.

Safety boundary

Keep the ESP32 side low-voltage. Unplug USB before rewiring. If the project interacts with mains power, motors, pumps, batteries, or outdoor wiring, use an isolated relay/module and get adult or qualified supervision.

Verification method

First confirm the input on GPIO5 in Serial Monitor. Then confirm the output pin GPIO25 changes only when the condition is true. This separates sensor problems from output-driver problems.
