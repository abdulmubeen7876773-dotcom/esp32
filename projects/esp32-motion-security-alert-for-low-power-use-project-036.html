ESP32 Motion Security Alert with Pir Motion for Shop Front

Security Projects · Advanced · 14 min read · Project

This hands-on Security project reads Pir Motion Sensor signal on GPIO33 and switches Buzzer control automatically.

This tutorial walks through motion security alert with pir motion for shop front — wiring, firmware, and a quick bench test.

You'll connect Pir Motion Sensor signal to GPIO33, configure a threshold, and drive Buzzer control on GPIO25.

Built for safe room: extend later with NTP Timestamp if you need remote visibility.

Project Overview

A beginner-friendly Security circuit where Pir Motion Sensor signal feeds the ESP32 and Buzzer control handles the physical action.

  • Pir Motion Sensor signal connects to GPIO33 (ADC input)
  • Threshold is set to 2060 in firmware (adjust after calibration)
  • Buzzer control is driven on GPIO25 when the condition is met
  • The main loop samples every 1949 ms while powered

How It Works

01

Read

The ESP32 samples the input sensor on GPIO33, producing a raw reading.

02

Compare

That reading is checked against a threshold configured in the sketch.

03

Decide

When the condition is met, the ESP32 sets GPIO25 high; otherwise it stays low.

04

Act

The ESP32 drives Buzzer control on GPIO25 high while the condition stays true, then releases it.

Code

Upload the following sketch to your ESP32 board using the Arduino IDE. Adjust pins and threshold for your hardware.

esp32_motion_security_alert_for_low_powe.ino
// ESP32 Motion Security Alert with Pir Motion for Shop Front
const int SENSOR_PIN = 33;
const int OUTPUT_PIN = 25;
const int THRESHOLD = 2267;
const int SAMPLE_DELAY_MS = 1949;

void setup() {
  Serial.begin(115200);
  pinMode(OUTPUT_PIN, OUTPUT);
  pinMode(SENSOR_PIN, INPUT);
  Serial.println("Starting: motion security alert with pir motion for shop front");
  Serial.print("Threshold: ");
  Serial.println(THRESHOLD);
}

void loop() {
  int reading = analogRead(SENSOR_PIN);
  Serial.print("Pir Motion Sensor signal: ");
  Serial.println(reading);
  bool active = reading > THRESHOLD;
  digitalWrite(OUTPUT_PIN, active ? HIGH : LOW);
  delay(SAMPLE_DELAY_MS);
}
