ESP32 Motion Security Alert with Buzzer Control for Office Door

Security Projects · Advanced · 14 min read · Project

Build a practical Window Alert setup where ESP32 monitors Pir Motion Sensor signal and controls Buzzer control.

This tutorial walks through motion security alert with buzzer control for office door — wiring, firmware, and a quick bench test.

You'll connect Pir Motion Sensor signal to GPIO33, configure a threshold, and drive Buzzer control on GPIO25.

Built for window alert: extend later with Local Server if you need remote visibility.

Project Overview

The firmware keeps logic local on the ESP32, making it dependable for Window Alert prototypes.

  • Pir Motion Sensor signal connects to GPIO33 (ADC input)
  • Threshold is set to 1991 in firmware (adjust after calibration)
  • Buzzer control is driven on GPIO25 when the condition is met
  • The main loop samples every 494 ms while powered

How It Works

01

Read

The ESP32 samples the input sensor on GPIO33, producing a raw reading.

02

Compare

That reading is checked against a threshold configured in the sketch.

03

Decide

When the condition is met, the ESP32 sets GPIO25 high; otherwise it stays low.

04

Act

Buzzer control activates on GPIO25 for each threshold crossing — watch the Serial log to confirm timing.

Code

Upload the following sketch to your ESP32 board using the Arduino IDE. Adjust pins and threshold for your hardware.

esp32_motion_security_alert_with_wifi_co.ino
// ESP32 Motion Security Alert with Buzzer Control for Office Door
const int SENSOR_PIN = 33;
const int OUTPUT_PIN = 25;
const int THRESHOLD = 2198;
const int SAMPLE_DELAY_MS = 494;

void setup() {
  Serial.begin(115200);
  pinMode(OUTPUT_PIN, OUTPUT);
  pinMode(SENSOR_PIN, INPUT);
  Serial.println("Starting: motion security alert with buzzer control for office door");
  Serial.print("Threshold: ");
  Serial.println(THRESHOLD);
}

void loop() {
  int reading = analogRead(SENSOR_PIN);
  Serial.print("Pir Motion Sensor signal: ");
  Serial.println(reading);
  bool active = reading > THRESHOLD;
  digitalWrite(OUTPUT_PIN, active ? HIGH : LOW);
  delay(SAMPLE_DELAY_MS);
}
