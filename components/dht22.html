DHT22 Temperature & Humidity Sensor

A pocket-sized weather probe for your ESP32 — it feels how warm and how damp the air is, then sends those numbers to your code. Perfect for room monitors, greenhouses, and your first real sensor mission.

🧒Explain Like I'm 12

Imagine a tiny weather reporter living inside a plastic box on your breadboard.

It checks the air around it — how warm it feels and how sticky or dry it is — then whispers two numbers to your ESP32 over one wire. Your ESP32 passes those numbers to your laptop, like a text message that says "24 degrees and a bit humid."

You do not need to understand every chip inside to use it. Wire three pins, run the code, and you already built something real — the same kind of sensor used in classroom weather stations and hobby greenhouses.

What would a curious 12-year-old ask next? "Can I blow on it and watch the numbers change?" Yes — and that is one of the best tests that your circuit actually works.

⚡Quick Facts

  • Measures Air temperature (°C) and relative humidity (%)
  • Pins on a module 3 — power, data, ground
  • Talks to ESP32 on One data pin (plus 3.3 V and GND)
  • Best for Room monitors, weather stations, plant care
  • Update speed About one fresh reading every 2 seconds
  • Accuracy (typical) ±0.5 °C temperature · ±2 % humidity
  • Logic level 3.3 V — matches ESP32 GPIO directly
  • First mission Read Temperature with DHT22 (Mission 2)

📋Technical Specifications

Arduino library: DHT sensor library by Adafruit (install via Arduino Library Manager — search "DHT sensor library")

  • What it does — The DHT22 measures temperature and relative humidity in the air around the sensor. Relative humidity (RH) means how full the air is with water vapor compared to the maximum it could hold at that temperature — like how full a sponge is with water.
  • How it works — Inside the package, a small sensor element changes with heat and moisture. A tiny chip converts those changes into digital numbers and sends them out on one DATA wire using a special timing pattern your ESP32 reads with a library.
  • Inside, simply — Think of two mini-detectors (one for heat, one for moisture) plus a translator chip. The translator packs temperature and humidity into a short burst of on/off pulses on the DATA line. Your ESP32 listens, decodes, and hands you friendly numbers.
  • Voltage — Modules are usually labeled 3.3–5 V on VCC. For direct wiring to ESP32 GPIO, use 3.3 V on VCC so the DATA line stays 3.3 V logic. Many breakout boards include a pull-up resistor; bare 4-pin sensors need a 10 kΩ pull-up from DATA to 3.3 V.
  • Communication — Single-wire digital protocol (not I2C or SPI). One GPIO pin handles all sensor traffic. The Adafruit DHT library hides the timing details.
  • Temperature range — −40 °C to 80 °C (typical datasheet range)
  • Humidity range — 0–100 % RH
  • Sampling — Wait at least 2 seconds between reads (0.5 Hz maximum)
  • DHT22 vs DHT11 — DHT22 is more accurate and better for learning projects you will trust. DHT11 is cheaper and updates faster but less precise. Most ESP32 Engine missions use DHT22.
  • Real-world uses — Desk weather displays · classroom climate logs · greenhouse alerts · checking if a bedroom feels too dry · smart-fan or dehumidifier triggers · data logging before adding Wi-Fi
  • Interesting facts — Breathe gently on the sensor and humidity jumps within seconds · move it near a sunny window and temperature drifts · the white grid on some sensors is the sensing area — avoid covering it with tape · the same sensor family is often labeled AM2302
  • Safety notes — Low-voltage 3.3 V breadboard project only · unplug USB before rewiring · keep liquids away from the board · do not touch the exposed sensor grid while powered · ask an adult for help with USB and small parts

📍Pinout

  • VCC (+) Power ESP32 3.3 V Use 3.3 V for direct ESP32 wiring. Red wire on many modules.
  • DATA (out) Signal Any ESP32 GPIO (example GPIO4) One wire carries all readings. Match this GPIO in your code. Yellow or green wire on many modules.
  • GND (−) Ground ESP32 GND Common ground with the ESP32. Black wire on many modules.
  • NC (4-pin bare sensors only) Not connected Leave unconnected Some bare DHT22 packages have four pins — the NC pin is not used on a module.

🔗Wiring

A three-pin DHT22 module is the friendliest way to start. You only need power, ground, and one data wire to the ESP32.

Use the same GPIO number in your code as on the breadboard. Unplug USB before moving wires.

  1. 1

    Unplug USB. Place the ESP32 and a half-size breadboard side by side.

  2. 2

    Plug the DHT22 module into the breadboard so the pins are not shorted.

  3. 3

    Connect VCC on the module to 3.3 V on the ESP32.

  4. 4

    Connect GND on the module to GND on the ESP32.

  5. 5

    Connect DATA on the module to GPIO4 on the ESP32 (you may use another GPIO — just change the code to match).

  6. 6

    Double-check — 3.3 V to VCC, GND to GND, DATA to the GPIO you chose.

  7. 7

    Plug in USB, open Serial Monitor at 115200 baud, and upload the example code below.

⌨️Code Example

Copy into Arduino IDE. Install the library listed above first.

dht22_read.ino 
#include "DHT.h"

#define DHTPIN 4
#define DHTTYPE DHT22

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(115200);
  dht.begin();
  Serial.println("DHT22 ready — readings every 2 seconds");
}

void loop() {
  float temp = dht.readTemperature();
  float humidity = dht.readHumidity();

  if (isnan(temp) || isnan(humidity)) {
    Serial.println("Read failed — check wiring and wait 2 s between reads");
  } else {
    Serial.printf("Temp: %.1f C  Humidity: %.0f%%\n", temp, humidity);
  }

  delay(2000);
}

✨Expected Output

Open Serial Monitor at 115200 baud. After upload you should see:

DHT22 ready — readings every 2 seconds Temp: 24.3 C Humidity: 55% Temp: 24.3 C Humidity: 55%

Every 2 seconds a new line appears. Breathe gently near the sensor — humidity should rise. Hold it near a window or warm laptop — temperature should shift.

If you see "Read failed", do not panic. That message means your code ran but the sensor did not answer — almost always wiring, wrong GPIO, or reading too fast.

⚠️Common Mistakes

  • Reading faster than every 2 seconds — the DHT22 needs time to prepare a fresh sample.
  • DATA wire on GPIO4 in code but connected to a different pin on the breadboard.
  • Using 5 V on VCC while assuming DATA is safe for ESP32 — prefer 3.3 V for direct GPIO wiring.
  • Forgetting a pull-up resistor on bare (non-module) sensors — DATA needs a 10 kΩ pull-up to 3.3 V.
  • Loose jumper wires that look connected but are sitting beside the pin, not in the hole.
  • Covering the sensor grid with tape or foam — airflow helps realistic readings.
  • Trusting the first reading instantly after power-on — wait one full 2-second cycle.

🔧Troubleshooting

  • Serial shows "Read failed" or NaN every time Unplug USB. Re-seat VCC, GND, and DATA. Confirm `#define DHTPIN` matches your wire. Install Adafruit DHT library. Wait 2 seconds between reads. Try another GPIO if the pin is damaged.
  • Worked once, then only NaN Check for a loose DATA jumper. Move the ESP32 and breadboard without tugging wires. Avoid GPIO strapping pins (GPIO0, GPIO2, GPIO12, GPIO15) if upload behaves oddly.
  • Temperature looks right but humidity is always 0 or 100 Let the sensor sit in open air for two minutes. Do not blow directly on it during the test. Swap a suspected bad module if values never move.
  • Humidity jumps wildly when I walk past Normal near body heat and breath. For stable classroom demos, mount the sensor away from the keyboard fan and give it 60 seconds to settle.
  • Numbers never change at all Verify 3.3 V at VCC with a multimeter if you have one. Confirm GND is shared. Re-upload the sketch and reset the board with the EN button.
  • Library compiles but upload fails Select the correct ESP32 board and COM port in Arduino IDE. Close Serial Monitor during upload. See the Install Arduino IDE guide linked below.
